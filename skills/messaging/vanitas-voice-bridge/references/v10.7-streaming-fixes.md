# v10.7 Streaming Fixes (2026-06-16)

Two critical bugs discovered and fixed in voice_agent_v10_7.py during Groq integration testing.

## Bug 1: `aiter_lines()` Infinite Hang

### Symptom
Proxy returns HTTP 200 (streaming accepted). Voice agent log shows NO "Vanitas" response. Session eventually disconnects after ~90s. All subsequent utterances buffered silently (is_responding=True never clears).

### Root Cause
`httpx.AsyncClient.stream().aiter_lines()` has no chunk-level timeout. If upstream SSE stream stalls (provider hang, network drop), the async iterator blocks the event loop indefinitely.

### Fix (voice_agent_v10_7.py, `_handle_reply_inner`)
```python
async def _stream_with_timeout():
    nonlocal full_reply
    async with httpx.AsyncClient(transport=TR, timeout=45) as c:
        async with c.stream("POST", PROXY_URL, json=payload) as r:
            if r.status_code != 200:
                raise Exception(f"Proxy {r.status_code}")
            last_chunk = asyncio.get_event_loop().time()
            async for line in r.aiter_lines():
                if not line.startswith("data: "): continue
                # parse token...
                if tok:
                    full_reply += tok
                    last_chunk = asyncio.get_event_loop().time()
                if asyncio.get_event_loop().time() - last_chunk > 20:
                    raise Exception("Stream stalled (20s no chunk)")

await asyncio.wait_for(_stream_with_timeout(), timeout=45)
```

### Recovery
If `full_reply` has partial content on timeout, use it instead of discarding.

### Discovery Timeline
- 18:17:16 "Yani birazcık" request sent, proxy 200 OK
- 18:17:18-18:17:22 New utterances arrive, buffered (is_responding=True)
- 18:18:54 Browser disconnect after 97s silence
- No Vanitas response logged for either request
- Previous 2 exchanges worked perfectly (first one 7s, second 5s)

## Bug 2: asyncio.Lock Recursive Deadlock

### Symptom
Voice agent stops responding after first multi-turn exchange. No errors. WebSocket stays open.

### Root Cause (v10.4-v10.6)
```python
async def handle_reply(text):
    async with reply_lock:        # Lock acquired
        try:
            await _handle_reply_inner(text)
        finally:
            if utterance_queue:    # Queued during response
                await handle_reply(full_text)  # DEADLOCK!
```
`asyncio.Lock` is NOT reentrant. Recursive call tries to acquire same lock.

### Fix
Move queue processing outside lock scope:
```python
async def handle_reply(text):
    async with reply_lock:
        try:
            await _handle_reply_inner(text)
        finally:
            is_responding = False
    # Lock released — safe to recurse
    if utterance_queue:
        full_text = " ".join(utterance_queue)
        utterance_queue = []
        await handle_reply(full_text)
```

### Why Missed
Single-turn tests have empty utterance_queue → no recursive call → no deadlock.
