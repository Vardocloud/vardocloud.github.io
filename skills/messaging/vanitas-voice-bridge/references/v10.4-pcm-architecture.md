# v10.4 PCM Architecture — Technical Deep Dive

## The Format War: Why v10-v10.2 Failed

### MediaRecorder `audio/webm;codecs=opus` (FAILED)
```
Browser: MediaRecorder(stream, {mimeType: 'audio/webm;codecs=opus'})
↓ ondataavailable
Each chunk: [EBML header] [Segment] [Cluster] [SimpleBlock: opus data] [BlockGroup]
                                                                  ↑ ONLY THIS is opus
                                                                  Everything else is WebM container
↓ WebSocket → Server → Deepgram `encoding=opus`
Deepgram: "I expected raw opus packets. You sent me WebM. corrupt data."
```

### AudioContext ScriptProcessor PCM (WORKS)
```
Browser: AudioContext({sampleRate: 16000}) + ScriptProcessor(1024, 1, 1)
↓ onaudioprocess
inputBuffer.getChannelData(0) → Float32Array[-1.0 ... 1.0]
↓ Float32 → Int16 conversion
Int16Array → raw bytes (16-bit little-endian PCM)
↓ WebSocket → Server → Deepgram `encoding=linear16&sample_rate=16000`
Deepgram: "Raw PCM at 16kHz mono. I understand this perfectly."
```

## Key Parameters (v10.4)

| Parameter | Value | Why |
|-----------|-------|-----|
| sample_rate | 16000 | Speech standard, balances quality/bandwidth |
| channels | 1 | Mono for STT |
| buffer_size | 4096 | 256ms chunks at 16kHz — fewer packets, less WebSocket overhead |
| encoding | linear16 | Universally supported, zero ambiguity |
| interim_results | true | Captures partial sentences for better UX |
| utterance_end_ms | 1000 | ⚠️ Minimum = 1000. 500 rejects with HTTP 400. Docs claim 1-20000 range but Nova-2 enforces 1000 floor. |
| silence_flush | 0.6s | Server-side: silence gap before flushing queued utterances to Hermes (reduced from 1.2s for lower latency) |
| tts_speed | 1.07 | Slightly faster than natural, not rushed |

## Proxy v2 Personality Injection

### Problem
Hermes `/v1/chat/completions` is stateless. It receives `{messages: [...], model: "..."}` and passes to the model provider. No SOUL.md, no MEMORY.md, no USER.md is loaded.

### Solution
Proxy v2 strips ALL incoming system messages and injects a single comprehensive Vanitas prompt containing:
- Full personality traits (from SOUL.md)
- Conversation rules (echo → question → listen, no interviews)
- Edel's profile (DAÜ, NEU, LinkedIn, privacy, iteration style)
- Memory facts (preferences, conventions)

### Why not use Hermes session?
Hermes session works through messaging gateways (Telegram, Discord). The `/v1/chat/completions` API is a different path — designed for stateless completions, not session-based conversation. For session memory in voice, the voice agent would need to integrate with Hermes's messaging system.

## Utterance Queuing Logic

**⚠️ Python scope pitfall (2026-06-16):** The `utterance_timer` variable MUST appear in the `nonlocal` declaration of `process_deepgram()`. Because `utterance_timer = asyncio.create_task(...)` is an ASSIGNMENT, Python treats it as local to the nested function. Without `nonlocal`, reading it before assignment crashes with `cannot access local variable 'utterance_timer' where it is not associated with a value`.

```python
async def process_deepgram():
    nonlocal utterance_queue, last_audio_time, utterance_timer  # ← ALL THREE required!
    ...
```

```python
# On each final transcript from Deepgram:
utterance_queue.append(transcript)
last_audio_time = time.time()
# Cancel previous timer, start new 1.2s countdown
if utterance_timer:
    utterance_timer.cancel()
utterance_timer = asyncio.create_task(delayed_flush(0.6))  # 0.6s silence → send

# delayed_flush:
await asyncio.sleep(0.6)
if time.time() - last_audio_time >= 0.5:  # No new audio arrived in 0.5s
    full_text = " ".join(utterance_queue)
    utterance_queue = []
    send_to_hermes(full_text)
```

This prevents mid-sentence splitting: if the user speaks continuously with <0.6s pauses, all their sentences are concatenated into one Hermes request.

## Model Routing (v10.4 Final — 2026-06-16)

### Evolution
1. **v10.4 initial**: voice agent → proxy → Hermes API (8642) → Pollinations
2. **Problem**: Hermes `/v1/chat/completions` ignores `model` field — routes ALL to Mistral. Bad Turkish.
3. **Fix**: proxy → Pollinations direct (19999) with `model=gemma`

### Current path
```
voice agent → proxy (8767) → Pollinations proxy (19999) → gen.pollinations.ai → gemma
```
- Pollinations proxy (19999) handles auth internally (reads POLLINATIONS_API_KEY from .env)
- No auth needed from our proxy — just POST JSON
- Model hardcoded to `gemma` in proxy — ignores voice agent's model param

### Why gemma
Tested 2026-06-16: gemma > gpt-5.4-mini > minimax > openai > glm for Turkish voice.
- gemma: warm, conversational, natural "sen," invites dialogue
- openai (Mistral): cold, formal "siz," lists bullet points
- glm: BROKEN — outputs internal reasoning

### Invalid model names
- `openai/gpt-4.1-nano` → Hermes timeout (20s) — doesn't exist on ANY endpoint
- `deepseek-v4-flash` → Hermes routes to Mistral anyway
- Hermes API model field is effectively ignored

## Deepgram Streaming API Reference

### Supported encodings (WebSocket)
| Encoding | Requires | Compatible with |
|----------|----------|-----------------|
| `linear16` | Raw PCM 16-bit LE | AudioContext PCM ✅ |
| `opus` | Raw opus packets | NOT MediaRecorder ❌ |
| `mulaw` | 8-bit μ-law | Rarely used |
| `alaw` | 8-bit A-law | Rarely used |

No WebM/OGG/MP3 container support in streaming API.

### Sample rates
8000, 16000, 24000, 32000, 44100, 48000
16000 is standard for speech recognition.
