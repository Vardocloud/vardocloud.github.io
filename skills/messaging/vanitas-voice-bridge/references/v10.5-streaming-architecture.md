# v10.5 Streaming Architecture — Walkie-Talkie'den Akıcı Sohbete

**Date:** 2026-06-16
**Predecessor:** v10.4 (utterance lock + buffering)
**Key insight from Edel:** "bu mantık voice agent akıcı konuşma mantığına oturmuyor"

## Problem: v10.4 Walkie-Talkie Model

```
User speaks → 0.6s silence → flush → LLM (non-streaming, 5-20s) → 
  full TTS generation → play audio → during playback, buffered speech → 
  after audio ends, process buffer
```

**Total dead silence:** 5-23 seconds between user speech and response start.
**No interruption:** User speaks during TTS → buffered → processed later.

## Solution: v10.5 Streaming + Sentence TTS + Interruption

```
User speaks → 0.4s silence → flush → LLM (streaming, first token 1-2s) →
  each sentence boundary (.!?…) → immediate TTS → play →
  user interrupts → stop audio ⚡ → process new speech immediately
```

## Architectural Changes

### 1. Streaming LLM (was: non-streaming)
```python
# v10.4: Wait for full response
r = await c.post(PROXY_URL, json={"stream": False})
reply = r.json()["choices"][0]["message"]["content"]

# v10.5: Stream tokens as they arrive
async with c.stream("POST", PROXY_URL, json={"stream": True}) as r:
    async for line in r.aiter_lines():
        if line.startswith("data: "):
            token = json.loads(line[6:])["choices"][0]["delta"]["content"]
```

### 2. Sentence-by-sentence TTS (was: whole response at once)
```python
# Accumulate tokens until sentence boundary
sentence_buf += token
if re.search(r'[.!?…\n]', token) or len(sentence_buf) > 100:
    # Send sentence text to frontend immediately
    await safe_send(json.dumps({"type": "assistant", "text": sentence}))
    # Generate and send TTS audio for just this sentence
    audio = await tts_sentence(sentence)
    await safe_send(audio)
```

### 3. Audio Interruption (was: no interruption)
```python
# Detect user speaking during LLM/TTS
if user_speaking:
    cancel_current = True
    reply_task.cancel()  # Cancel current LLM streaming
    await safe_send(json.dumps({"type": "interrupt"}))  # Frontend stops audio

# Frontend JavaScript:
function stopCurrentAudio() {
    if (currentAudio) { currentAudio.stop(); currentAudio = null; }
}
// On 'interrupt' message: stopCurrentAudio()
```

### 4. Global Cancel Flag (new in v10.5)
```python
cancel_current = False  # Set True when user interrupts

# Checked at every stage:
if cancel_current: break  # During LLM token iteration
if cancel_current: break  # Before TTS generation
if cancel_current: break  # Before sending audio
```

### 5. User Speaking Detection (simplified from v10.4)
```python
user_speaking = False

# In process_deepgram:
if transcript or is_final:
    user_speaking = True
    last_active = time.time()

# After speech_final or 3s silence:
user_speaking = False
```

## Key Differences from v10.4

| Aspect | v10.4 | v10.5 |
|--------|-------|-------|
| LLM mode | Non-streaming | Streaming |
| TTS timing | After full response | Per sentence |
| First audio | 5-23s after speech | 2-4s after speech |
| Interruption | Buffered, delayed | Immediate, cancels |
| Cancel mechanism | None | `cancel_current` flag |
| Sentence splitting | N/A | Regex `[.!?…\n]` |
| Frontend audio | One blob | Multiple sequential |
| User_speaking | Implicit via utterance | Explicit flag |
| Silence threshold | 0.6s | 0.4s |
| Deepgram utterance_end_ms | 1000 | 800 |

## Critical Implementation Details

### Transcript Always Visible (bug fix from v10.4)
```python
async def flush_utterances():
    full_text = " ".join(utterance_queue).strip()
    # ALWAYS send transcript to frontend FIRST
    await safe_send(json.dumps({"type": "transcript", "text": full_text}))
    # THEN check if we should process (skip LLM if responding)
    if is_responding:
        return  # Keep in queue, will process after reply done
    utterance_queue = []
    await handle_reply(full_text)
```

### No Double-Transcript
The `handle_reply` wrapper directly calls `handle_reply(full_text)` for post-reply buffered utterances — NEVER calls `flush_utterances()` which would re-send the transcript.

### Timeout Chain (v10.5)
```
Voice agent → Proxy: 60s streaming (was 20s non-streaming)
Proxy → Hermes: 60s streaming (was 25s)
Voice agent TTS per sentence: 12s (was 15s)
```

### cancel_current Checkpoints
Must check `cancel_current` at:
1. Start of each SSE line iteration
2. Before TTS generation
3. After TTS generation (before sending audio)
4. Before sending remaining buffer text
