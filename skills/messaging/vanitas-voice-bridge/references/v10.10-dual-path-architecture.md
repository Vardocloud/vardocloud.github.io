## Fast Path (Default)
- **Provider:** Groq direct API (api.groq.com)
- **Model:** `meta-llama/llama-4-scout-17b-16e-instruct`
- **Prompt:** `soul_core.md` (626 bytes) — compressed personality
- **Memory:** `memory_index.json` — grep-based retrieval, key facts only
- **Wiki:** `grep -rl` over `~/wiki/` — file name matching only
- **First token:** ~151ms (streaming)
- **Max tokens:** 150

## Deep Path (Triggered)
- **Provider:** Hermes Gateway (port 8642) via proxy (port 8767)
- **Model:** DeepSeek V4 Pro (full SOUL.md + MEMORY.md + USER.md)
- **Prompt:** Full 22KB system context + thinking
- **First token:** 2-3 seconds

## Soniox STT Integration
- **URL:** `wss://stt-rt.soniox.com/transcribe-websocket` (SDK-verified)
- **Model:** `stt-rt-v5`
- **Audio:** PCM s16le, 16kHz, mono
- **Language hints:** `["tr"]`
- **Endpoint detection:** enabled, 1000ms max delay
- **Flush delay:** 2.5 seconds silence before LLM call

### Critical Soniox Pitfalls
1. **`is_final` is per-token**, not per-message. Check `t.get("is_final")` on each token.
2. **Subword tokens** need `"".join()` not `" ".join()` for Turkish
3. **Final tokens are cumulative** — use `final_buffer` to deduplicate
4. First message sent to WebSocket must be JSON config, then raw audio bytes
