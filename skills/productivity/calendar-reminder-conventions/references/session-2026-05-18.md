# 2026-05-18 — Reminder timing failure

## What happened
Edel had a calendar event at 07:00. Vanitas sent a reminder at 08:30 — 1.5 hours AFTER the event started.

Root cause: profile note said "Bugün cafe mesai (17-23)" which was about Sunday May 17. Vanitas incorrectly inferred that meant "tomorrow morning (Monday) also has cafe work" — profile inference overriding actual calendar data.

## What Edel corrected
- Calendar event reminder time should be read from Google Calendar's actual reminder offset
- If Edel set 1 hour reminder → use 1 hour, not the default 30 min
- Profile notes are past/recurring context, never authoritative for future events
- Calendar data always wins
