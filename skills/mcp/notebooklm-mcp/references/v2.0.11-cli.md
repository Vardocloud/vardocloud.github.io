# notebooklm-mcp v2.0.11 — CLI Reference (CURRENT)

## Overview

`notebooklm-mcp` v2.0.11 replaces the old `nlm` CLI (v0.8.1 jacob-bd). The CLI was consolidated — there is no separate `nlm` binary for NotebookLM. All operations go through the MCP server or the `notebooklm-mcp` binary directly.

## Available Commands

```
notebooklm-mcp [OPTIONS] COMMAND [ARGS]...
```

### Global Options

| Option | Description |
|--------|-------------|
| `--version` | Show version and exit |
| `-c, --config PATH` | Configuration file path |
| `--debug` | Enable debug logging |
| `--help` | Show help message |

### Commands

| Command | Description |
|---------|-------------|
| `chat` | Interactive chat with NotebookLM |
| `config-show` | Show current configuration |
| `export-profile` | Export Chrome profile for sharing |
| `import-profile` | Import existing Chrome profile |
| `init NOTEBOOK_URL` | Initialize MCP Server with notebook URL |
| `quick-setup` | Quick setup with config file and optional profile import |
| `server` | Start the FastMCP v2 NotebookLM server |
| `test` | Test connection to NotebookLM |

### test (Auth Verification)

```bash
notebooklm-mcp test -n <notebook_id>
notebooklm-mcp test -n <notebook_id> --headless
```

Opens a browser (or uses headless mode), navigates to the notebook URL, and reports auth status.

- **Success:** ✅ Browser started → ✅ Authentication valid → ✅ Navigated to notebook
- **Failure:** ⚠️ Authentication required — redirected to Google sign-in

### init (First-Time Setup + Auth)

```bash
notebooklm-mcp init https://notebooklm.google.com/notebook/<notebook_id>
notebooklm-mcp init <notebook_id>
notebooklm-mcp init <notebook_url> --headless
notebooklm-mcp init <notebook_url> -o /path/to/config.yaml
```

Opens browser for Google login (or headless), saves auth cookies and generates config.

### server (Start MCP)

```bash
notebooklm-mcp server
```

Starts the FastMCP v2 server. Used by Hermes MCP integration. Reads auth from `~/.notebooklm-mcp-cli/auth.json`.

### config-show

```bash
notebooklm-mcp config-show
```

Displays current configuration and auth status.

### chat (Interactive)

```bash
notebooklm-mcp chat
```

Starts an interactive chat session with NotebookLM.

### quick-setup

```bash
notebooklm-mcp quick-setup -c /path/to/config.yaml --import-profile
```

One-command setup with config file and optional Chrome profile import.

### export-profile / import-profile

```bash
notebooklm-mcp export-profile -o /path/to/profile.tar.gz
notebooklm-mcp import-profile /path/to/profile.tar.gz
```

Export/import Chrome profiles for sharing auth between machines.

## What's Missing vs v0.8.1

The following old `nlm` subcommands do **NOT** exist in v2.0.11:

| Old Command (v0.8.1) | Status in v2.0.11 |
|----------------------|-------------------|
| `nlm login --check` | ❌ — use `notebooklm-mcp test -n <id>` |
| `nlm doctor` | ❌ — use `notebooklm-mcp test -n <id>` |
| `nlm notebook list` | ❌ — use MCP tool `notebook_list` |
| `nlm notebook get` | ❌ — use MCP tool `notebook_get` |
| `nlm source add` | ❌ — use MCP tool `source_add` |
| `nlm studio create` | ❌ — use MCP tool `studio_create` |
| `nlm download audio` | ❌ — use MCP tool `download_artifact` |
| `nlm query` | ❌ — use MCP tool `notebook_query` |
| `nlm batch` | ❌ — use MCP tool `batch` |
| `nlm pipeline` | ❌ — use MCP tool `pipeline` |

All these operations are available via the MCP server's tool set (39 tools). You just can't call them directly from the CLI anymore — they require an MCP session.

## Auth Store

- **Location:** `~/.notebooklm-mcp-cli/auth.json`
- **Format:** JSON array of cookie objects
- **Managed by:** Keepalive Chrome (CDP port 18800) + nb_keepalive.py
